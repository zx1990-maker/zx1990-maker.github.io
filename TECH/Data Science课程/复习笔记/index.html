    1. Complexity Ranking

    常见时间复杂度从低到高：

    Big-O 中文说明 English
    O(1) 常数 constant
    O(log n) 对数 logarithmic
    O(n) 线性 linear
    O(n log n) 线性对数 linearithmic
    O(n^2) 平方 quadratic
    O(2^n) 指数 exponential

    Array

    Operation Big-O Notes
    arr[i] O(1) 通过 index 随机访问

    4. Rules for Simplifying Big-O

    规则 1：忽略常数倍

    for i in range(100 * n):
    ...

    复杂度：O(n)。

    规则 2：只保留最高阶

    O(n + n^2) -> O(n^2)

    规则 3：顺序循环相加，最后简化

    for i in range(n):
    ...

for j in range(n):
    ...

    复杂度：O(n + n) -> O(n)。

    注意：如果两个循环是嵌套关系，才是 O(n^2)。

    6. 常考的 Big-O Pattern

    Pattern 1：外层是常数，内层是 n

    def func1(n):
    int_sum = 0
    for i in range(123):      # 外层 O(1)
        for j in range(n):    # 内层 O(n)
            int_sum += 1

    复杂度：O(n)。

    虽然是双层循环，但外层只跑固定次数 123，所以不是 O(n^2)。

    Pattern 2：三角形 nested loop

    def func2(n):
    for i in range(n):
        for j in range(i, n):
            ...

    次数约为：

    n + (n - 1) + (n - 2) + ... + 1
= n(n + 1) / 2

    复杂度：O(n^2)。

    Pattern 3：对数步数

    def model1(n):
    i = 1
    while i < n:
        i *= 2

    假设 while 执行 k 次：

    i = 2^k
2^k >= n
k >= log n

    复杂度：O(log n)。

    Pattern 4：外层线性，内层对数

    def model2(n):
    total = 0
    for i in range(n):        # O(n)
        j = 1
        while j < n:          # O(log n)
            total += 1
            j *= 2

    复杂度：O(n log n)。

    Pattern 5：调和级数 / 步长为外层变量

    def model3(n):
    for i in range(1, n + 1):
        for j in range(1, n + 1, i):
            ...

    内层次数大约是：

    n/1 + n/2 + n/3 + ... + n/n = n ln n

    复杂度：O(n log n)。

    Pattern 6：几何增长的内层总和

    def model4(n):
    total = 0
    i = 1
    while i < n:              # 外层 O(log n)
        for j in range(i):    # 1, 2, 4, 8, ...
            total += 1
        i *= 2

    虽然外层是 O(log n)，但内层总次数为：

    1 + 2 + 4 + ... + n = O(n)

    所以总复杂度：O(n)。

    6. Binary Search

    Binary search 的前提：array must be sorted。

    def binary_search(arr, target, low, high):
    # base case
    if low > high:
        return None

    mid = (low + high) // 2

    if target == arr[mid]:
        return mid
    elif target < arr[mid]:
        return binary_search(arr, target, low, mid - 1)
    else:
        return binary_search(arr, target, mid + 1, high)

    复杂度：O(log n)。

    明确：

    Step 2：Base Case

    11. Backtracking Template

    def backtrack(path):
    if len(path) == size:
        result.append(path[:])
    else:
        for choice in choices:
            # 做出选择
            path.append(choice)

            # recurse
            backtrack(path)

    def binary_search(self, element):
    lo = 0
    hi = self.nb_elts - 1

    while lo <= hi:

    for i in range(n):

    i = 0
j = n - 1

while i < j:
    total = arr[i] + arr[j]

    if total < target:
        i += 1
    elif total > target:
        j -= 1
    else:
        return i, j

    seen = {}
for i, x in enumerate(arr):

        return seen[complement], i
    seen[x] = i

    复杂度：O(n)。

    1. Stack 栈

    1.1 定义

    Stack = LIFO

    Last In, First Out
    后进先出 / 先进后出

    例子：

    push(1)
push(2)
push(3)

top -> 3
       2
       1

    pop() 会移除最上面的元素，也就是当前的 top。

    1.2 Stack Operations

    Operation Meaning Complexity
    push(x) 压栈，把元素放到 top O(1)
    pop() 弹栈，移除并返回 top O(1)
    peek() 查看 top，不删除 O(1)
    is_empty() 判断栈是否为空 O(1)

    1.3 Stack Implementation

    Python list 可以直接模拟 stack：

    stack = []

stack.append(x)   # push
stack.pop()       # pop
stack[-1]         # peek

    1.4 Stack Applications

    Parentheses Matching 括号匹配

    判断括号是否正确配对。

    例子：

    ()[]{}  -> True
([)]    -> False

    思路：

    1. 遇到左括号，push 到 stack。
    2. 遇到右括号，检查 stack top 是否匹配。
    3. 如果不匹配，返回 False。
    4. 最后 stack 必须为空。
    def is_valid_parentheses(s):
    stack = []
    pairs = {
        ")": "(",
        "]": "[",
        "}": "{",
    }

    for char in s:
        if char in "([{":
            stack.append(char)
        else:
            if not stack:
                return False

            top = stack.pop()

            if pairs[char] != top:
                return False

    return len(stack) == 0

    复杂度：

    Time:  O(n)
Space: O(n)

    Monotonic Stack 单调栈

    单调栈用于维护一个单调递增或单调递减的栈。

    核心思想：

    • 当新元素破坏单调性时，不断 pop。
    • 每个元素最多入栈一次、出栈一次。
    • 因此总复杂度是 O(n)。

    常见用途：

    • Next Greater Element
    • Next Smaller Element
    • Largest Rectangle in Histogram
    • Stock Span Problem

    2. Queue 队列

    2.1 定义

    Queue = FIFO

    First In, First Out
    先进先出

    例子：

    enqueue(1)
enqueue(2)
enqueue(3)

head -> 1  2  3 <- tail

dequeue() removes 1

    2.2 Queue Operations

    Operation Meaning Complexity
    enqueue(x) 入队，从 tail 加入 O(1)
    dequeue() 出队，从 head 删除 O(1)
    peek() 查看 head，不删除 O(1)
    is_empty() 判断是否为空 O(1)

    2.3 Queue Structure

    队列通常维护两个指针：

    Pointer Meaning
    head / front 删除位置
    tail / rear 插入位置

    2.4 Circular Queue 循环队列

    普通 array queue 的问题：

    dequeue: O(n)

    因为删除第一个元素后，后面的元素需要向前移动。

    循环队列的解决方法：

    • 不移动元素。
    • 只移动 head 和 tail 指针。
    • 使用取模 % 让指针回到数组开头。
    head = (head + 1) % capacity
tail = (tail + 1) % capacity

    复杂度：

    enqueue: O(1)
dequeue: O(1)

    2.5 Queue Application: BFS

    Queue 最常见的应用是 Breadth-First Search。

    BFS 的特点：

    1. 先访问距离为 1 的节点。

    2. Simple Sort

    2.1 Bubble Sort 冒泡排序

    核心思想：

    • 比较相邻元素。
    • 如果顺序错误，就 swap。
    • 每一轮会把最大值移动到当前未排序部分的最后。
    • 如果某一轮没有发生 swap，说明数组已经有序，可以提前结束。
    def bubble_sort(A):
    n = len(A)

    for i in range(n - 1):
        swapped = False

        for j in range(n - i - 1):
            if A[j] > A[j + 1]:
                A[j], A[j + 1] = A[j + 1], A[j]
                swapped = True

        if not swapped:
            break

    复杂度：

    Best:    O(n)
Average: O(n²)
Worst:   O(n²)

    2.2 Selection Sort 选择排序

    核心思想：

    • 每一轮从未排序部分找到最小值。
    • 把最小值 swap 到当前 front。
    • 已排序部分不断扩大。
    def selection_sort(A):
    n = len(A)

    for i in range(n - 1):
        min_idx = i

        for j in range(i + 1, n):
            if A[j] < A[min_idx]:
                min_idx = j

        A[i], A[min_idx] = A[min_idx], A[i]

    复杂度：

    Best:    O(n²)
Average: O(n²)
Worst:   O(n²)

    特点：

    Not stable

    2.3 Insertion Sort 插入排序

    核心思想：

    • 左边部分始终保持有序。
    • 每次取下一个元素。
    • 把它插入左边有序部分的正确位置。
    def insertion_sort(A):
    for i in range(1, len(A)):
        temp = A[i]
        j = i - 1

        while j >= 0 and A[j] > temp:
            A[j + 1] = A[j]
            j -= 1

        A[j + 1] = temp

    复杂度：

    Best:    O(n)
Average: O(n²)
Worst:   O(n²)

    特点：

    • 如果数组 almost sorted，Insertion Sort 会很快。
    • 移动元素，而不是每次都 swap。
    • Stable。

    2.4 Merge Sort 归并排序

    核心思想：

    1. Split array into two halves.
    2. Recursively sort left half.
    3. Recursively sort right half.
    4. Merge two sorted halves.

    Merge Two Sorted Lists

    def merge(A, B):
    result = []
    i, j = 0, 0

    while i < len(A) and j < len(B):
        if A[i] <= B[j]:
            result.append(A[i])
            i += 1
        else:
            result.append(B[j])
            j += 1

    result += A[i:]
    result += B[j:]

    return result

    Merge Sort

    def merge_sort(A):
    if len(A) <= 1:
        return A

    mid = len(A) // 2

    left = merge_sort(A[:mid])
    right = merge_sort(A[mid:])

    return merge(left, right)

    复杂度：

    Best:    O(n log n)
Average: O(n log n)
Worst:   O(n log n)

    空间复杂度：

    O(n)

    特点：

    • 需要 extra array，所以不是 in-place。
    • 如果两个元素相等时优先取 left array 的元素，则 stable。

    5. Quick Sort 快速排序

    5.1 Core Idea

    Quick Sort 的核心思想：

    1. 选一个 pivot。
    2. 把数组 partition 成两部分：
    3. 左边：<= pivot
    4. 右边：>= pivot
    5. 递归排序左半边和右半边。

    笔记中使用的是 median-of-three 选 pivot：

    从左、中、右三个元素中选中位数，减少 bad pivot 出现的概率。

    5.2 Median of Three

    def median_of_three(S, a, b):
    mid = (a + b) // 2

    temp = [S[a], S[mid], S[b]]
    temp.sort()

    S[b] = temp[1]      # median as pivot
    S[a] = temp[0]
    S[mid] = temp[2]

    这里把三者中的中位数放到 S[b]，作为 pivot。

    5.3 In-place Quick Sort

    def quick_sort(A, a, b):
    if a >= b:
        return

    median_of_three(A, a, b)

    pivot = A[b]
    left = a
    right = b - 1

    while left <= right:
        while left <= right and A[left] <= pivot:
            left += 1

        while left <= right and A[right] > pivot:
            right -= 1

        if left <= right:
            A[left], A[right] = A[right], A[left]
            left += 1
            right -= 1

    A[left], A[b] = A[b], A[left]

    quick_sort(A, a, left - 1)
    quick_sort(A, left + 1, b)

    5.4 Partition 过程总结

    1. 先选 pivot。
    2. left 找 > pivot 的元素。
    3. right 找 <= pivot 的元素。
    4. 如果 left <= right，交换两个元素。
    5. 指针交叉后，把 pivot 放到最终位置。
    6. 对 pivot 左右两边分别递归。

    5.5 Quick Sort Complexity

    Case Complexity
    Best O(n log n)
    Average O(n log n)
    Worst O(n²)

    空间复杂度：

    Case Space
    Average O(log n)
    Worst O(n)

    原因：

    • 递归调用会产生 call stack。
    • 如果 pivot 选择很差，每次只分出一个元素，会退化成 O(n²)。

    5.6 Quick Sort 特点

    • 平均情况很快。
    • 但 bad pivot 可能导致 O(n²)。
    • 不是 stable sort。
    • median-of-three 是为了降低选到极端 pivot 的概率。

    6. Shell Sort

    6.1 Core Idea

    Shell Sort 是 gap-based insertion sort。

    核心思想：

    1. 先用较大的 gap 做 insertion sort。
    2. 让离得较远的元素可以更快移动。
    3. 然后逐渐缩小 gap。
    4. 最后一轮 gap = 1，等价于普通 insertion sort。

    6.2 Shell Sort Code

    def shell_sort(A):
    n = len(A)
    gap = n // 2

    while gap > 0:
        for i in range(gap, n):
            temp = A[i]
            j = i - gap

            while j >= 0 and A[j] > temp:
                A[j + gap] = A[j]
                j -= gap

            A[j + gap] = temp

        gap //= 2

    6.3 Shell Sort 特点

    • 最后一轮一定是 gap = 1，也就是 insertion sort。
    • 因为前面已经用大 gap 做过粗略排序，最后一轮 insertion sort 会更快。
    • Not stable：因为 gap move 可能改变相等元素的相对顺序。

    7. Radix Sort 基数排序

    7.1 Core Idea

    Radix Sort 用 bucket 根据数字的每一位进行排序。

    常见做法：

    1. 从 least significant digit 开始，也就是个位。
    2. 每一位使用 10 个 buckets：0–9。
    3. 每一轮按当前 digit 分桶。
    4. 再按 bucket 顺序收集回数组。
    5. 直到最高位处理完。

    7.2 Determine Digit

    def determine_digit(integer, digit):
    return integer // (10 ** digit) % 10

    例子：

    determine_digit(472, 0)  # 2, ones digit
determine_digit(472, 1)  # 7, tens digit
determine_digit(472, 2)  # 4, hundreds digit

    7.3 Radix Sort Code

    from math import log10
from collections import deque

def radix_sort(A):
    buckets = [deque() for _ in range(10)]

    max_val = max(A)
    digit = 0
    total_passes = int(log10(max_val)) + 1

    while digit < total_passes:
        for value in A:
            d = determine_digit(value, digit)
            buckets[d].append(value)

        index = 0

        for b in range(10):
            while buckets[b]:
                A[index] = buckets[b].popleft()
                index += 1

        digit += 1

    return A

    7.4 Radix Sort Complexity

    设：

    Symbol Meaning
    n number of elements
    d number of digits
    b number of buckets, decimal case usually 10

    复杂度：

    Time:  O(d(n + b))
Space: O(n + b)

    如果 b = 10，通常写作：

    O(dn)

    7.5 Radix Sort 特点

    • 从 least significant digit 开始。
    • Bucket 必须保持 stable。
    • 因此 bucket 通常用 queue：先进先出。
    • Radix Sort 不是 comparison-based sorting。
    • 需要知道数字有多少 digit。
    • 当 digit 数量较小的时候很快。

    def insert(node, v):
    if node is None:
        return BSTNode(v)

    if v < node.value:
        node.left = insert(node.left, v)
        node.left.parent = node

    elif v > node.value:
        node.right = insert(node.right, v)
        node.right.parent = node

    else:
        node.count += 1

    return node

    • 插入后仍然要保持 BST property。

    15. BST Delete

    删除 BST node 时分三种情况。

    15.1 Case 1: Node is Leaf

    当前 node 没有 child。

    处理方式：

    直接删除
parent.left 或 parent.right = None

    15.2 Case 2: Node Has One Child

    当前 node 只有一个 child。

    处理方式：

    用 child 代替当前 node
child.parent = node.parent

    示意：

    Before:
parent -> node -> child

After:
parent -> child

    15.3 Case 3: Node Has Two Children

    当前 node 有两个 children。

    处理方式：

    1. 找到当前 node 的 successor。
    2. 用 successor 的值替换当前 node。
    3. 删除原来的 successor。

    原因：

    successor 是 right subtree 中最左边的 node

    它一定是比当前 node 大的最小值，所以可以安全替换当前 node。

    15.4 Delete 总结

    Case Condition Solution
    Case 1 Leaf node Directly delete
    Case 2 One child Replace node by child
    Case 3 Two children Replace by successor, then delete successor

            current = current.right

        return None

    root = TreeNode(root_value)

    root_index = inorder.index(root_value)

    left_inorder = inorder[:root_index]

    right_preorder = preorder[1 + left_size:]

    root.left = build_tree(left_preorder, left_inorder)
    root.right = build_tree(right_preorder, right_inorder)

    return root

    2.2 BFS - Breadth-First Search

    BFS 使用：

    Queue

    特点：

    一层一层访问

    BFS Code

    def bfs(g, v):
    output = str(v)
    c = [WHITE] * g.nb_vertices()
    q = []

    q.append(v)
    c[v] = GREY

    while q != []:
        i = q.pop(0)

        for j in g.out_edges(i):
            if c[j] == WHITE:
                q.append(j)
                output += str(j) + " "
                c[j] = GREY

        c[i] = BLACK

    return output

    BFS 过程

    1. 把起点放进 queue。
    2. 从 queue 中取出最早加入的 vertex。
    3. 访问它的所有 unvisited neighbors。
    4. 把这些 neighbors 加入 queue。
    5. 重复直到 queue 为空。

    BFS Complexity

    Time:  O(V + E)
Space: O(V)

    BFS 常见用途

    • Shortest path in unweighted graph
    • Level order traversal
    • Connected components
    • 判断是否可以到达某个 node

    如果 vertex i 可以通过某条 path 到达 vertex j，
那么结果 graph 中应该有 edge i -> j。

    这就是：

    Transitive Closure

    3.2 核心逻辑

    如果：

    i -> k
k -> j

    那么可以推出：

    i -> j

    所以添加 edge：

    gnew.add_edge(i, j)

    3.3 Warshall-style Code

    def computeConnectivity(g):
    result = g.nb_vertices()
    gnew = copy.deepcopy(g)

    for k in range(result):
        for i in range(result):
            for j in range(result):
                if (
                    gnew.has_edge(i, k)
                    and gnew.has_edge(k, j)
                ):
                    gnew.add_edge(i, j)

    return gnew

    3.4 三层循环解释

    for k:

    把 k 当作中间点。

    for i:

    起点。

    for j:

    终点。

    如果从 i 到 k，又从 k 到 j，则说明 i 可以到 j。

    设 vertex 数量为 V：

    Time:  O(V³)

    4. Spanning Tree

    4.1 题目含义

    computeMinimumSpanningTree(g, root) 的目标：

    从 root 出发，生成一棵包含 reachable vertices 的 spanning tree。

    注意：

    这里笔记中的版本更像是 traversal-based spanning tree，不是带权图中的 Kruskal / Prim 那种真正的 minimum spanning tree。

    4.2 Spanning Tree 思路

    1. 从 root 开始。
    2. 遍历 graph。
    3. 每次第一次发现一个新 vertex，就把对应 edge 加入新 graph。
    4. 最后得到一棵不重复访问节点的 tree。

    4.3 Code

    def computeMinimumSpanningTree(g, root):
    graph_type = type(g)
    gnew = graph_type(g.nb_vertices())

    c = [WHITE] * g.nb_vertices()
    stack = []

    stack.append(root)

    while stack != []:
        i = stack.pop()

        if c[i] == WHITE:
            c[i] = GREY

            for j in g.out_edges(i):
                if c[j] == WHITE:
                    gnew.add_edge(i, j)
                    stack.append(j)

            c[i] = BLACK

    return gnew

    4.4 为什么会有不同的 Spanning Tree？

    同一个 graph 可能有多个 spanning tree。

    原因：

    • 遍历顺序不同。
    • neighbor 顺序不同。
    • 可选择的 edge 不止一条。

    因此，同一个 graph 可以得到不同的 spanning tree。

    Collision 指：

    发生 collision 时，不换位置，而是在同一个 index 下面接一个 list。

    1 94 -> 39
    2 empty
    3 empty
    4 empty
    5 44 -> 88 -> 11
    6 empty
    7 empty

    9 16 -> 5

    Chaining 特点

    优点：

    • table 不一定会因为单个 slot 被占用而插入失败。

    缺点：

    • 如果很多 key 落在同一个 bucket，最坏情况会退化为 O(n)。

    核心思想

    如果 h(k) 的位置被占了，就往后一个一个找：

    h(k)
h(k) + 1
h(k) + 2
h(k) + 3
...

    需要对 table size 取模：

    index = (h(k) + step) % table_size

    Value 13 94 39 16 5 44 88 11 12 23 20

    Linear Probing 特点

    优点：

    • 不需要额外 linked list。

    缺点：

    • 容易产生 primary clustering。
    • 一段连续占用的位置会越来越长。
    • 查找可能变慢。

    7.3 Quadratic Probing

    核心思想

    如果 h(k) 被占用，不是一步一步找，而是按平方跳：

    h(k) + 1²
h(k) + 2²
h(k) + 3²
...

    对 table size 取模：

    index = (h(k) + step ** 2) % table_size

    原因：

    def put(self, k, v):
    new_item = KVItem(k, v)
    h = self._hash_function(k)
    bucket = self._table[h]

    if new_item in bucket:
        i = bucket.index(new_item)
        bucket[i] = new_item
    else:
        bucket.append(new_item)
        self._size += 1

    解释：

    • 先算 hash index。
    • 找到对应 bucket。
    • 如果 key 已存在，更新 value。
    • 如果 key 不存在，append 新 item。

    for item in self._table[h]:
        if item._key == k:

    def remove(self, k):
    h = self._hash_function(k)

    for item in self._table[h]:
        if item._key == k:
            self._table[h].remove(item)
            self._size -= 1
            return item._value

    return None
