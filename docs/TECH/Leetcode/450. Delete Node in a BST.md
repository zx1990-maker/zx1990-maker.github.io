# 450. Delete Node in a BST
**题目链接：** https://leetcode.com/problems/delete-node-in-a-bst/description/
## 题目描述：
Given a root node reference of a BST and a key, delete the node with the given key in the BST. Return the root node reference (possibly updated) of the BST.

Basically, the deletion can be divided into two stages:

1. Search for a node to remove.

2. If the node is found, delete the node.

## Solution：
```Python
class Solution(object):
    def deleteNode(self, root, key):
        """
        :type root: Optional[TreeNode]
        :type key: int
        :rtype: Optional[TreeNode]
        """
        # 1. Base Case
        if root is None: # Python 规范推荐用 is None
            return None
        
        # 2. 分治搜索阶段：利用返回值自动接线
        if key < root.val:
            root.left = self.deleteNode(root.left, key)
        elif key > root.val:
            root.right = self.deleteNode(root.right, key)
            
        # 3. 命中目标！直接改成 else，逻辑闭环
        else:
            # 情况 A：叶子节点（两边都为空，直接返回 None 让上层父节点接住并物理断开）
            if root.left is None and root.right is None:
                return None 
            # 情况 B：只有右子树（打包托付右子树大部队）
            elif root.left is None:
                return root.right
            # 情况 C：只有左子树（打包托付左子树大部队）
            elif root.right is None:
                return root.left
            
            # 情况 D：极其强壮的双子树
            else:
                succ = root.right
                while succ.left is not None:
                    succ = succ.left
                
                root.val = succ.val # 偷梁换柱
                # 精准降维：让右子树自己去刨掉那个替身值
                root.right = self.deleteNode(root.right, succ.val)
                
        return root
```
## 逻辑梳理：
### 🎯 要素一：明确函数“输入”与“输出”的语义 (Define Semantics)

这是整段代码最成功、最核心的立足点。在动笔前，必须在脑海中达成铁一般的契约与承诺：

* **输入 (Input)**：当前子树的根节点 `root`（以及要删除的 `key`）。
* **输出 (Output)**：**“删除了 `key` 之后，这棵子树的全新根节点。”**

因为死死咬住了这个输出语义，整个代码的所有出口都保持了 100% 的纯洁性：
1. **空树时**：返回 `None`（符合语义）。
2. **删掉自己时**：返回孩子节点（符合语义，因为孩子成了新根）。
3. **处理完当前层后**：最后一行返回 `root`（符合语义，代表处理好的树根）。

---

### 🚪 要素二：确定 Base Case（安全出口）

写下的第一段拦截：

```python
if root == None:
    return  # 在 Python 中，这代表返回 None
```
逻辑语义：如果在全树中一路深入，最后撞到了树的尽头（None），说明要删除的 key 在这棵树里根本不存在。

安全弹栈：此时既然不存在，树的拓扑结构没有任何改变，直接把 None 像安全气囊一样扔回去，开始触发弹栈。

### 🛠 要素三：编写当前层的合并逻辑（利用子问题返回值）
这一要素在代码中展现得淋漓尽致，可以完美划分为两个阶段：“高管委派阶段（分治搜索）” 和 “现场决策阶段（命中删除）”。

#### 阶段 A：高管委派（利用子问题返回值更新物理连接）
```Python
if key < root.val:
    root.left = self.deleteNode(root.left, key)
elif key > root.val:
    root.right = self.deleteNode(root.right, key)
```
你坐在当前 root 的办公室里。如果 key 比你小，说明要删的目标在你的左子树里：

完全信任：你直接调用 self.deleteNode(root.left, key)。由于你 100% 信任要素一的承诺，你知道这个调用执行完毕后，一定会交还给你一棵“已经完美删除了 key 的全新左子树”。

等号挂载（核心）：你用 root.left = ... 像接线员一样，用等号稳稳接住下属递上来的新树根。此时，不论下属在底层经历了多么惊心动魄的指针断开，在当前层看来，只是用一个等号刷新了左侧的物理连线。

#### 阶段 B：现场决策（命中目标 key == root.val）
当程序走到第三个分支时，意味着你（当前 root）就是那个被判了死刑、必须被抹去的节点。你立刻开始清点身家（分析子树情况）：

##### 情况 1：孤家寡人（叶子节点）
```Python
if root.left == None and root.right == None:
    return None
```
你发现自己既没有左孩子，也没有右孩子。你对自己说：“我走之后，我的位子留给 None。”

你直接 return None。上一层调用你的父节点，就会用刚才阶段 A 里的 root.left = ...（或 .right）稳稳接住这个 None。在物理层面上，你的父节点主动松开了连向你的手，你成功在内存中隐退。

##### 情况 2 & 3：打包托付（单边子树）
```Python
elif root.left == None:
    return root.right
elif root.right == None:
    return root.left
```
你发现自己只有一个孩子（例如只有右孩子）。如果你死了，你的右孩子必须继承你的王位。

你潇洒地执行 return root.right。

注意：右孩子下面可能连着千军万马、成百上千个孙子辈节点。但由于你直接把 root.right 这根总线指针扔给了上一层父节点，父节点用等号一接，整棵右子树就顺次整体平移了上去！绝不会丢失任何一个孙子，也不需要任何手动连线。

##### 情况 4：降维打击（极其强壮的双子树）
```Python
else:
    succ = root.right
    while succ.left != None:
        succ = succ.left
    root.val = succ.val
    root.right = self.deleteNode(root.right, succ.val)
```
当你发现左右子树都极度强壮时，你遇到了整道题最难的拓扑死结。但你用两步精妙的操作实现了降维打击：

偷梁换柱：你顺着右子树一路向左，找到了后继节点（右子树里最小的替身 succ）。你把它的数值抄了过来（root.val = succ.val）。这样，你就不用去改动极其复杂的、原本连向你的那些指针了！你只是换了个马甲，树的物理拓扑结构稳如泰山。

派发刺杀任务：树里现在出现了两个值等于 succ.val 的节点，必须把右子树深处的那个原本的替身叶子节点杀掉。
你转过身，对你的右子树办公室下达了精准命令：root.right = self.deleteNode(root.right, succ.val)。

💡 这行调用的绝妙之处：
它去右子树里删除 succ.val 时，因为 succ 节点是最左节点，它绝对不可能有左孩子（要么是叶子，要么只有右孩子）。这意味着，这一次向下推进，百分之百只会触发前面简单利落的“情况 1”或“情况 2”！那个深处的替身节点会被瞬间物理抹去，并用等号把刷新后的右子树完美缝合回来。
