# 110. Balanced Binary Tree
**题目链接：** https://leetcode.com/problems/balanced-binary-tree/description/
## 题目描述：
Given a binary tree, determine if it is height-balanced.
## Solution：
```Python
# Definition for a binary tree node.
# class TreeNode(object):
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution(object):
    def isBalanced(self, root):
        """
        :type root: Optional[TreeNode]
        :rtype: bool
        """
        
        def get_height(node):
            """
            核心三要素定义：
            【要素一：明确语义】
            输入一个节点：
            - 如果它是健康的、平衡的，返回这棵子树的【真实高度（节点数）】；
            - 如果它深处的任何一个角落已经失衡了，什么都不用管，直接返回【 -1 】作为全网警报。
            """
            
            # 【要素二：Base Case 安全出口】
            # 空树天然是健康的、平衡的，高度（节点数）为 0，安全返回 0
            if node is None:
                return 0
            
            # 【要素三：当前层合并逻辑】
            # 1. 信任子问题，先派左边去自查
            left_height = get_height(node.left)
            if left_height == -1: 
                return -1  # 左子树已经拉响警报，高管层不需要再看右边，直接向董事会汇报 -1
                
            # 2. 再派右边去自查
            right_height = get_height(node.right)
            if right_height == -1: 
                return -1  # 右子树已经拉响警报，直接向董事会汇报 -1
            
            # 3. 此时左右子树都健康，轮到当前节点做最后的现场裁决
            # 检查左右两边汇报上来的高度差，绝对值是否超过了 1
            if abs(left_height - right_height) > 1:
                return -1  # 在我这一层发生严重倾斜，立刻触发警报，返回 -1
            else:
                # 完美通过安检！履行要素一的契约：计算并上报当前子树的真实高度
                return max(left_height, right_height) + 1

        # 主流程：只要全树从底向上自查完，最顶层拿到的结果不是 -1，就代表整棵树完全平衡！
        return get_height(root) != -1
        ```
